# 아우룸 맥북 Obsidian 원격 연결 장애 분석 및 해결 보고서

- **작성일:** 2026-06-05
- **대상 장비:** 아우룸 맥북 (`aurum-macbookpro`, `100.115.9.49`)
- **원격 서버:** 아톰 서버 (`gigabyte-atom`, `100.98.149.127`)
- **작업자:** AI 개발 및 인프라 운영 에이전트

## 1. 장애 현상 및 증상

1. **원격 한글 파일/폴더 실종:**
   - Obsidian에서 새로 만든 파일은 서버에 정상 저장되지만, 기존 서버에 존재하던 한글 명칭의 파일/폴더(예: `001. LLM 설정`, `000. 워크스페이스 인덱스.md` 등) 및 영어 파일 일부가 Obsidian 탐색기에서 누락되거나 읽히지 않음.
2. **원격 연결 완전 끊김 (좀비 마운트):**
   - Obsidian 앱 구동 시 `ENXIO: no such device or address, watch '/Users/nams/Obsidian/dgx_workspace/'` 에러 팝업 발생 후 앱 로딩 실패.
   - 이후 조치 시도 시 `ENOTCONN: socket is not connected` 및 `Device not configured` 에러가 터미널과 앱에서 지속해서 발생.

## 2. 상세 원인 분석

### ① 한글 자소 분리 (NFD vs NFC) 인코딩 문제
- **Linux(서버):** 한글 파일명을 초성/중성/종성이 결합된 **NFC** 형식으로 저장합니다.
- **macOS(로컬):** 한글 파일명을 초성/중성/종성이 각각 분리된 **NFD** (UTF-8-MAC) 형식으로 처리합니다.
- **문제점:** 기존 `sshfs` 연결 시 인코딩 매핑 옵션이 누락되어, Mac의 Obsidian이 NFD로 파일 정보를 요구하면 리눅스 서버에서 인식을 못 하고 `No such file or directory` 에러를 반환했습니다.

### ② 무부하 캐시 방지 옵션으로 인한 SSHFS 백엔드 데몬 크래시
- 캐시를 완전히 끄고 마운트(`cache=no, attr_timeout=0`)한 결과, Obsidian과 같은 전단 탐색 도구가 폴더를 읽어 들일 때 무수히 많은 미세 메타데이터(Stat) 요청이 발생했습니다.
- 과도한 통신 부하로 인해 백그라운드의 `sshfs` 데몬이 연결을 끊거나 강제 종료(Crash)되면서 마운트 테이블만 남고 연결은 끊어진 **좀비 마운트(ENXIO / ENOTCONN)** 상태로 변질되었습니다.

## 3. 해결책 및 적용 내용

프로그램 충돌이 잦고 안정성이 떨어지는 3rd-party 드라이버(`sshfs` + `macFUSE`) 대신, **macOS 네이티브 파일 공유 프로토콜인 SMB(Samba)**를 연계하는 아키텍처로 변경했습니다.

```text
[아톰 서버 (Linux)]
       │ (Samba 서비스 구동)
       ▼ (Tailscale VPN 대역: 100.98.149.127)
[아우룸 맥북 (macOS)]
       │ 1. 네이티브 SMB 마운트 (NFC/NFD 자동 변환 지원)
       ▼ [경로: ~/caiser77_mount]
       │ 2. 심볼릭 링크 생성 (경로 포인팅)
       ▼ [경로: ~/Obsidian/dgx_workspace]
[Obsidian Vault 앱]
```

### 적용된 주요 기술적 조치
1. **마운트 백엔드 전환:** `sshfs`를 강제 해제하고 macOS의 파일 시스템 수준에서 유니코드 변환 및 재연결을 완벽하게 처리해 주는 네이티브 **SMB 마운트**로 전환했습니다.
2. **구조적 심볼릭 링크 매핑:** Obsidian 내의 Vault 설정 주소를 변경할 필요 없이 기존 경로(`~/Obsidian/dgx_workspace`)를 그대로 가리킬 수 있도록 `~/caiser77_mount/dgx_workspace`로의 소프트 링크를 매핑했습니다.

## 4. 아우룸 맥북 마운트 복구 및 장기 유지 가이드

맥북을 종료하거나 재부팅하여 연결이 끊겼을 때, 아래 방법으로 손쉽게 원클릭 복구할 수 있습니다.

### 방법 A: macOS Finder를 이용한 그래픽(GUI) 복구 (강력 추천)
1. 아우룸 맥북 화면에서 **`Cmd + K`** 단축키를 눌러 "서버에 연결" 창을 엽니다.
2. 서버 주소 입력창에 아래 주소를 입력하고 **`연결`**을 클릭합니다.
   ```text
   smb://100.98.149.127/caiser77
   ```
3. 계정 정보 입력:
   - **사용자 이름:** `caiser77`
   - **비밀번호:** 별도 보안 채널로 관리되는 SMB 계정 비밀번호
   - *팁: "이 암호를 키체인에 기억"에 체크해 두시면 다음부터 패스워드 입력 과정도 생략됩니다.*

### 방법 B: 맥북 터미널 명령어를 이용한 복구
터미널을 열고 다음 한 줄의 명령어를 복사하여 붙여넣고 엔터를 치면 즉시 마운트가 복원됩니다.
```bash
mkdir -p ~/caiser77_mount
mount_smbfs '//caiser77@100.98.149.127/caiser77' ~/caiser77_mount
```
