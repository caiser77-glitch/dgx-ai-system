
# [시스템 설계] 생태 데이터 자동화 대시보드 구축 계획

본 문서는 제공된 문헌조사 및 현지조사 데이터를 통합하여 생태 데이터 분석을 위한 웹 대시보드를 구축하기 위한 **시스템 설계 원칙, 데이터 모델, 분석 로직, 그리고 시각화 전략**을 정의합니다. 이 계획은 향후 Hermes Agent가 데이터를 자동으로 처리하고 분석하는 데 필요한 시스템 가이드라인으로 사용될 것입니다.

## 1. 데이터 통합 및 모델링 원칙 (Data Integration & Modeling Principles)

### 1.1. 분류군 동등 위상 처리 (Equal Hierarchy Processing)
모든 분류군(포유류, 조류, 어류 등)을 동등한 위상으로 처리하여 데이터 통합의 공정성을 확보합니다.

*   **계층적 통합:** 데이터는 가장 상위 레벨(예: Order)에서 시작하여 하위 레벨(Family, Species)로 계층적으로 구조화하여 저장합니다.
*   **평등한 가중치 부여:** 모든 분류군 데이터는 데이터베이스 레벨에서 동등한 가중치를 가지도록 설계하며, 특정 분류군에 대한 통계적 분석이 다른 분류군보다 우선하거나 덜 중요하지 않도록 합니다 [1].

### 1.2. 데이터 모델 설계 (Data Model Design)

데이터의 복잡성을 관리하고 복잡한 질의에 대응하기 위해 **관계형 데이터 모델(Relational Data Model)**을 기반으로 데이터를 저장합니다.

*   **핵심 테이블:**
    1.  **분류군 마스터 테이블:** 모든 분류군(Order, Family, Species)의 고유 ID와 이름 목록을 관리합니다.
    2.  **조사 결과 테이블:** 각 조사(문헌/현지) 결과를 분류군 및 개체 단위로 연결하여 저장합니다.
    3.  **통계 지표 테이블:** 모든 분류군에 대해 계산된 핵심 지표($H'$, $J'$, $R1$, $DI$, $eH'$)를 별도로 저장하여 빠른 검색과 비교를 지원합니다.

### 1.3. 데이터 통합 로직 (Data Integration Logic)

**통합 단계:**
1.  **정규화 (Normalization):** 모든 분류군 데이터를 표준화된 형식(예: 과학적 명명법)으로 통일합니다.
2.  **매핑 (Mapping):** 문헌조사 결과와 현지조사 결과를 **분류군 수준**에서 매핑하여 통합합니다.
3.  **지표 계산:** 각 분류군 또는 하위 분류군별로 종다양성($H'$)과 균등도($J'$)를 계산하여 통계 지표 테이블에 기록합니다.

## 2. 핵심 분석 지표 및 계산 수식 (Core Metrics & Formulas)

모든 분석의 기반이 되는 통계 지표의 계산 수식을 명확히 정의합니다.

### 2.1. 종다양성 (Shannon Index, $H'$)
종다양성은 생물 다양성을 측정하는 핵심 지표입니다.

$$H' = - \sum_{i=1}^{S} p_i \cdot \ln(p_i)$$

*   $S$: 해당 분류군 내의 전체 종(Species)의 수입니다.
*   $p_i$: $i$번째 종의 상대 빈도($n_i / N$)입니다.

### 2.2. 균등도 (Pielou's Evenness, $J'$)
균등도는 종 분포가 얼마나 균일하게 퍼져 있는지를 측정합니다.

$$J' = H' / \ln(S)$$

### 2.3. 예외 처리 전략 (Exception Handling Strategy)
데이터가 0인 경우에도 레이아웃이 깨지지 않도록 예외 처리를 시스템 레벨에서 강제합니다.

*   **분모 예외 처리:** $\ln(S)$ 계산 시, $S \le 1$인 경우 (종이 1개 이하인 경우) $\ln(S)$가 정의되지 않거나 오류가 발생하므로, 해당 경우 $\text{Pielou's Evenness}$($J'$)는 $\mathbf{1.0}$ 또는 $\mathbf{0.0}$으로 정의하여 통계적 오류를 방지합니다.
*   **데이터 부재 표시:** $\text{H'}$ 또는 $\text{J'}$ 값이 계산 불가능하거나 데이터가 없는 경우 (예: 조사 결과가 0일 때), 해당 셀에는 명시적으로 $\mathbf{NaN}$ 또는 $\mathbf{N/A}$ 대신 **"데이터 없음"** 또는 **"분석 불가"**와 같은 사용자 친화적인 메시지를 표시합니다.

## 3. 웹 대시보드 시각화 로직 (Visualization Logic for Streamlit/HTML)

데이터의 복잡성을 효과적으로 전달하기 위해 다중 시각화 방법을 사용합니다.

### 3.1. 막대 그래프 (Bar Chart)
*   **목표:** 분류군 간의 통계적 차이를 직관적으로 비교합니다.
*   **적용:** **분류군별 종다양성($H'$)** 또는 **균등도($J'$)** 값을 사용하여 막대 그래프를 생성합니다.
*   **특징:** $\text{H'}$ 값을 기준으로 내림차순 정렬하여, 가장 다양하고 균등한 분류군을 상단에 배치합니다.

### 3.2. 파이 차트 (Pie Chart)
*   **목표:** 특정 분류군 내부의 종 분포 균등도를 시각화합니다.
*   **적용:** 특정 분류군(예: 특정 Family)을 선택했을 때, 그 분류군 내의 각 종($p_i$)이 전체에 차지하는 상대적 비율을 파이 차트로 표현합니다.
*   **특징:** 이는 특정 분류군 내부의 미시적인 균등도에 초점을 맞추어, 전체 데이터의 맥락을 이해하는 데 도움을 줍니다.

### 3.3. 히트맵 (Heatmap)
*   **목표:** 다차원적인 데이터의 관계와 패턴을 한눈에 파악합니다.
*   **적용:** **상위 분류군(Order)과 하위 분류군(Family)**을 행과 열로 설정하고, 각 셀에 해당 조합의 **종다양성($H'$)** 또는 **균등도($J'$)** 값을 색상 농도로 매핑합니다.
*   **특징:** 복잡한 생물학적 관계 속에서 잠재적인 다양성 패턴의 클러스터링을 발견할 수 있도록 돕습니다.

## 4. 실행 요약 (Execution Summary for Agent)

Hermes Agent는 다음의 순서로 작업을 실행해야 합니다.

1.  **데이터 로드 및 정제:** `<source id="1">` 태그 내의 텍스트를 파싱하여 분류군, 종, 개체 수 데이터를 Pandas DataFrame으로 변환한다.
2.  **지표 계산:** 정의된 $H'$ 및 $J'$ 수식을 사용하여 모든 분류군 데이터에 대해 통계 지표를 계산한다. **(중요: $S \le 1$ 예외 처리 적용)**.
3.  **데이터 모델 구축:** 계산된 지표를 별도의 통계 테이블에 저장하여 관계형 데이터 모델을 완성한다.
4.  **시각화 생성:** Streamlit 또는 Matplotlib을 사용하여 다음을 생성한다.
    *   전체 데이터의 KPI 카드.
    *   분류군별 $H'$ 및 $J'$ 비교 막대 그래프.
    *   분류군 간의 $H'/J'$ 관계를 시각화하는 히트맵.
    *   사용자 선택에 따른 상세 종 분포 파이 차트.

이 시스템 설계는 데이터의 깊이와 넓이를 모두 고려하여, 단순한 결과 제시를 넘어 **탐색적인 분석**이 가능한 대시보드를 목표로 합니다.
