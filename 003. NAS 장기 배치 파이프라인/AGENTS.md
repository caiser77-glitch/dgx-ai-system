## 🛠️ 2. 핵심 실행 규칙 (Rules)
1. **서버 자원(VRAM) 보존 및 OOM 방지**:
   * vLLM 72B API 서버가 GPU 메모리의 90% 이상을 이미 점유하고 있으므로, 임베딩 모델(SentenceTransformer) 로드 시 **반드시 CPU 모드(`device='cpu'`)**로 동작해야 합니다. GPU OOM을 일으키지 않도록 사전에 리소스 검증 로직을 포함하십시오.
2. **무지성 일괄 처리 금지 (슬라이싱 정책)**:
   * NAS의 전체 파일(8.7만 개) 중 95% 이상을 차지하는 **현장 조사 사진(`.jpg`, `.png`)은 가공 대상에서 철저히 배제**하십시오.
   * 지식 추출 목적의 순수 문서 포맷(`pdf`, `hwp`, `hwpx`, `docx`, `xlsx`, `xls`)만 필터링하여 일자별로 **300 ~ 500개 내외**로 제한(Slicing)해서 구동하는 배치 스크립트 구조를 유지하십시오.
3. **인코딩 정규화(NFD/NFC) 예방**:
   * macOS NFD 자모 분리 이름 현상에 대응할 수 있도록 디렉토리 검색 및 파일 트래킹 시 와일드카드 매칭을 생활화하십시오.
4. **로깅 및 자동 복구 (Fail-safe)**:
   * 배치 가공 중 오류(vLLM API 타임아웃, OCR 서버 500 에러 등)가 발생한 파일은 완전히 스킵하지 않고, 상태 메타데이터(`metadata.json`)에 에러 코드를 마킹하여 다음 기동 시 재시도(Retry)할 수 있도록 설계하십시오.
5. **결과물 언어 규칙**:
   * 작성되는 모든 보고서, 소스 코드 내 주석, 상태 메시지는 **한국어**로 정제하여 제공되어야 합니다.
