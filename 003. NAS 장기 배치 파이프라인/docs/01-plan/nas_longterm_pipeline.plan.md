# 📋 [003. NAS 장기 배치 파이프라인] 구축 계획서 (Plan)

본 계획서는 아우룸생태연구소 NAS의 10개년 데이터(약 8.7만 개 파일)를 서버 부하와 리스크 없이 매일 안정적으로 요약 및 색인하여, 궁극적으로 RAG(지식 검색) 시스템에 점진적으로 반영하기 위한 **장기 배치 파이프라인 구축 프로젝트 계획서**입니다.

---

## 🎯 1. 프로젝트 목적 및 방향
* **목적**: 10년간 누적된 방대한 비정형 문서 자산을 일시에 처리하지 않고, 매일 밤 안정적인 분량만큼 점진적으로 가공 및 색인하는 **지속 가능한 배치 파이프라인**을 완성합니다.
* **핵심 방향**:
  * **서버 자원 보존**: vLLM 72B API 서버 및 로컬 OCR API의 과부하 및 OOM(VRAM 부족) 리스크 예방.
  * **지속 가능한 파이프라인**: 매일 밤 자동으로 구동(Cron) 및 증분 처리하여, 실패한 파일에 대한 예외 복구(Retry) 프로세스를 확립.
  * **RAG 검색 효율 극대화**: 중복 문서 제거, 쓸데없는 현장 사진(8만 개 이상) 필터링을 통해 순수 지식 문서만 정확히 임베딩.

---

## 📊 2. 대상 데이터셋 식별 및 필터링 전략
* **전체 파일 규모**: **86,986 개**
* **데이터 필터링 정책**:
  * **1순위 제외 (비대상)**: 사진 파일 (`.jpg`, `.png` 등 약 8.2만 개) 및 GIS 공간 데이터 (`.shp`, `.prj` 등), CAD 도면 (`.dwg`).
  * **2순위 대상 (핵심 문서)**: `pdf`, `hwp`, `hwpx`, `docx`, `xlsx`, `xls` (총 약 3.2만 개).
  * **배치 슬라이싱 처리량**: 하루 밤(00:00 ~ 05:00)당 **300 ~ 500개 파일** 한도로 순차 가공 진행.

## 📈 3. PDCA(Plan-Do-Check-Act) 마일스톤

### 🔍 [1단계] Plan (계획 수립 - 현재)
* **목표**: 10개년 대규모 배치 가공 범위 설정 및 하루 처리 용량 산정, DB 상태 테이블 관리 전략 수립.
* **산출물**: [nas_longterm_pipeline.plan.md](file:///Volumes/caiser77/dgx_workspace/003. 회사 NAS 10개년 데이터 분석/docs/01-plan/nas_longterm_pipeline.plan.md)
* **세부 태스크**:
  * NAS 디렉토리 트리 순회 순서 및 연도별 우선순위 수립.
  * 가공 이력(성공/실패/스킵/에러 코드)을 안전하게 기록할 SQLite 기반 메타데이터 DB 또는 상태 JSON 아키텍처 설계.

### 🛠️ [2단계] Do (구현 및 실행)
* **목표**: 일자별 슬라이서 구동 스크립트 작성 및 배치 스케줄러(Cron) 등록, 가상환경 연동.
* **산출물**: 
  * `scripts/longterm_batch_slicer.py` (하루 할당량 슬라이싱 및 분산 실행)
  * `scripts/cron_daily_pipeline.sh` (매일 밤 12시 기동 스크립트)
* **세부 태스크**:
  * 텍스트가 없는 스캔본 PDF의 선별적 OCR 호출 로직 고도화.
  * AI 요약 실패 시 Fallback 메커니즘 연동.

### 📊 [3단계] Check (점검 및 검증)
* **목표**: 일자별 작동 이력 모니터링, Slack webhook 장애 실시간 전송, 성공률 분석.
* **산출물**: `logs/daily_batch_summary.log` 및 Slack 장애 통보 모듈
* **세부 태스크**:
  * 매일 아침 전날 밤 기동된 배치 결과 보고서(성공 개수, OOM 발생 여부) 생성.
  * OCR API `500` 에러 파일들만 모아 수동 검증 큐로 이관.

### ⚙️ [4단계] Act (개선 및 최적화)
* **목표**: 병목 지점(vLLM API 속도 등) 개선, VRAM 점유 조절, 인덱스 자동 병합(Merge) 최적화.
* **세부 태스크**:
  * 대용량 FAISS 인덱스 병합 시 메모리 오버헤드 최소화 튜닝.
  * 중복 파일 해시 필터링 엔진 최적화.
