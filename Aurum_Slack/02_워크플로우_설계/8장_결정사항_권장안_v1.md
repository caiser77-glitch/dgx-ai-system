## 결정 1. 일지 자동 마감 시점

**옵션:**
- (a) 30분 무입력 시 자동 마감
- (b) `/마감` 명시 입력 시에만 마감
- (c) 둘 다 — 30분 무입력 또는 `/마감` 중 빠른 쪽

**권장: (c) 둘 다**
- 근거: 현장 조사는 이동·식사 등 자연 공백이 잦음. 30분 단독은 오마감 위험, `/마감` 단독은 잊어버림 위험. 두 조건 OR로 묶으면 안전망 됨.
- 보완: 자동 마감 5분 전 슬랙 DM으로 "5분 뒤 자동 마감됩니다. 추가 입력 없으면 종료" 알림.

**관리자 확인:** [ ] ✅ 동의 / [ ] ❌ 결정:

## 결정 2. 프로젝트 코드 부여 방식

**옵션:**
- (a) 슬랙 채널별 자동 (예: `#proj-시화-EIA` 채널에 올린 내용 = proj-시화-EIA)
- (b) 메시지 태그 (`#proj-xxx` 명시)
- (c) 두 가지 다 — 채널이 우선, 태그가 있으면 덮어쓰기

**권장: (c) 두 가지 다, 채널 우선 + 태그 덮어쓰기**
- 근거: 평소엔 채널만으로 충분 (조사원이 신경 안 써도 됨), 복수 프로젝트 동시 진행 시 태그로 구분 가능.
- 보완: 둘 다 없으면 일지를 "미분류"로 표시하고 관리자 검토 큐로 자동 이동.

**관리자 확인:** [ ] ✅ 동의 / [ ] ❌ 결정:

## 결정 3. 종 동정 신뢰도 처리

**옵션:**
- (a) LLM 단독 — 모든 출력 자동 신뢰
- (b) 의심 종은 자동으로 관리자 확인 요청
- (c) 모든 종을 관리자 검토 후 확정

**권장: (b) 의심 종은 관리자 확인 요청**
- 근거: 멸종위기종·천연기념물 오동정은 평가서 신뢰도에 직격탄. 관리자 100% 검토는 자동화 의미 상실.
- 트리거: `confidence: low` OR 멸종위기·천연기념물 추정 OR 조사자 메모에 "추정/?/불확실" 단어 포함
- 처리: 슬랙 #현장-조사 일지 게시 시 해당 종 옆에 ⚠️ 표시 + 관리자 DM "확인 요청"

**관리자 확인:** [ ] ✅ 동의 / [ ] ❌ 결정:

## 결정 4. 초안 게시 채널

**옵션:**
- (a) #현장-조사 직접 게시 (전체 노출)
- (b) 관리자 DM 후 승인 시 #현장-조사에 게시
- (c) 조사자 본인 + 관리자에게만 DM, 양쪽 확인 후 채널 게시

**권장: (a) #현장-조사 직접 게시 + ⚠️ 표시로 미확정 표기**
- 근거: 옵션 (b)·(c)는 관리자 병목 발생. 20명 규모에서 매일 일지 5~10건씩 들어오면 (b)·(c)는 관리자 휴가·출장 시 적체. 직접 게시 + 시각 표시가 더 운영적임.
- 게시 형식:
  - 모든 일지 초안은 헤더에 `[초안]` 표기
  - 관리자 ✅ 이모지 시 헤더가 `[확정]`으로 변경, NAS 확정 폴더로 이동
  - 관리자가 1주 안에 확정 안 한 일지는 자동 "잠정 확정" + #공지 채널에 주간 미확정 목록 자동 게시

**관리자 확인:** [ ] ✅ 동의 / [ ] ❌ 결정:

## 결정 5. 사진 보관 기한

**옵션:**
- (a) 무기한 보관 (원본 그대로)
- (b) 1년 후 저해상도(예: 1280px) 변환
- (c) 3년 후 저해상도 변환
- (d) 5년 후 저해상도 변환

**권장: (c) 3년 후 저해상도 변환**
- 근거:
  - 환경영향평가 후 사후 조사 의무 기간(보통 3~5년) 동안은 원본 필요
  - NAS 용량 압박은 3년치 누적 시점부터 본격적으로 옴
  - 저해상도(1920px·2MB 이하)로 변환해도 보고서·검토 용도엔 충분
- 예외: 멸종위기종·천연기념물 사진은 무기한 원본 보존 (별도 태그로 식별)
