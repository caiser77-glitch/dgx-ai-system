## 📐 고유 아키텍처 및 데이터 흐름 명세

### 1. 100% 결정론적 야장 데이터 정제 파이프라인
```
[1_입력_야장기록.txt] (원천 텍스트)
       │
       ▼ (swarm_orchestrator.py 정규표현식 파서)
[species_search_index.json] (학명 및 계통 보정 대조)
       │
       ├─────────────────────────────────┐
       ▼ (정량 정제 데이터 세이브)        ▼ (WGS84 위경도 지리 보정)
[parsed_data.json]               [생태현장_정밀공간좌표_정제완료.geojson]
       │                                 │
       └────────────────┬────────────────┘
                        ▼ (격식체 평서문 보전의견 합성 / API 타임아웃 예외 제어)
        [2_결과_최종보고서.md] & [2_결과_최종보고서.hwpx]
```

### 2. 공간 GIS 데이터 표준 좌표계 규칙 (Coordinates Standard)
* 생태현장의 멸종위기 야생생물 지리 공간 레이어 생성 시, 도분초(DMS) 및 한국 원점 좌표계를 WGS84 위경도 십진수로 변환해야 합니다.
* GeoJSON 표준 사양에 의거하여 coordinates의 배열 순서는 반드시 **`[경도, 위도]`** 순으로 올바르게 교차 기록되어야 지도의 오프셋 왜곡을 예방할 수 있습니다. (예: `[127.4851, 37.4912]`)

### 3. API 격리 예외 제어 및 격식체 폴백 (Fault-tolerance)
* **로컬 AI API 예외 격격**: 모델 헬스체크 및 LLM 의견 생성 API 호출 시, 타임아웃 시간을 `2초` 이내로 엄격히 통제하고, API 통신 에러 발생 시 파이프라인을 붕괴하지 않는 대신 사전에 조율된 격식체 보전의견(평서문 3문장)을 즉각 주입하여 무중단 완료를 보장해야 합니다.
* **보고서 격식성**: 생성되는 최종보고서 내 텍스트 서술 구조는 상위 `report_dna_analysis.md` 마스터 스타일 가이드를 동적으로 로드 및 주입하여 최고 권위의 환경영향평가 격식체를 고수해야 합니다.
* **생태 분류군 정렬 룰**: 보고서상 생물상 도표 렌더링 시 반드시 **`포유류 -> 조류 -> 어류 -> 양서파충류 -> 저서무척추`** 순으로 하향 정렬되도록 오케스트레이터가 계통을 분류해야 합니다.
