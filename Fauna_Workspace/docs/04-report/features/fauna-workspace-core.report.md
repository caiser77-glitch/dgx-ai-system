## 4. 학습한 점 및 회고 (Lessons Learned)

### 4.1 잘된 점 (What Went Well)
- **높은 수준의 내결함성(Fault-tolerance) 달성**: 로컬 AI 연동 및 외부 API 의존성은 언제든 통신 오류나 미구동 상태가 발생할 수 있습니다. 2초 타임아웃 예외 격리 처리를 구현하여 로컬 서버 상태에 전혀 지장받지 않는 무중단 100% 무결 구동 파이프라인을 확립한 부분이 매우 고무적입니다.
- **WGS84 [경도, 위도] 교차 순서 정합성 해결**: 지리학적 표준 GeoJSON의 coordinates 순서는 `[경도, 위도]`이지만 일반 표기는 `[위도, 경도]` 순이 많아 혼선을 줍니다. 이를 오케스트레이터가 사전 대조 단계에서 올바르게 교환 저장하여 중첩 융합 시 지도 렌더링 오차를 선제 방지하였습니다.

### 4.2 개선이 필요한 점 (What Needs Improvement)
- **코드 내부의 하드코딩 사전**: `swarm_orchestrator.py`에 계통 한글 명칭 치환 및 멸종위기종 좌표 대조사전이 일부 하드코딩되어 있습니다. 향후 마스터 인덱스 데이터베이스인 `species_search_index.json`에 이 딕셔너리 정보들을 통합해 외부 데이터셋 형태로 관리하면 엔진의 독립성을 한층 향상시킬 수 있을 것입니다.
